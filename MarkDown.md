## Resumen video y respuesta de las preguntas.

Se introduce la transición del desarrollo basado en terminales hacia las aplicaciones web. Explicando que el internet es una "red de redes" donde la información viaja en paquetes a través de routers siguiendo diversos caminos hasta su destino.
La comunicación se rige por el protocolo TCP/IP. El IP asigna direcciones únicas a cada dispositivo, evolucionando de IPv4 (32 bits) a IPv6 (128 bits) para satisfacer la demanda global de direcciones. Por su parte, el TCP gestiona servicios específicos mediante números de puerto (como el 80 para HTTP y 443 para HTTPS) y garantiza la entrega fiable de datos mediante la fragmentación y secuenciación de paquetes.
Se detallan servicios esenciales como el DNS, que traduce nombres de dominio en direcciones IP, y el DHCP, que asigna automáticamente configuraciones de red. Finalmente, se analiza el protocolo HTTP/HTTPS, utilizado para solicitar y recibir páginas web. Se examinan los componentes de una URL y los mensajes HTTP, destacando el método GET y los códigos de estado, tales como 200 OK (éxito), 301 (redirección) y 404 (no encontrado). El profesor demuestra cómo inspeccionar estos procesos usando curl y las Developer Tools del navegador
